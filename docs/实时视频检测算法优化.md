# 实时视频流检测算法设计

## 目标

识别家庭环境下具体是哪个/哪些物体发生了动作变化，并将其连续帧送给 VLM 进行场景判定。

---

## 流程

### 1. 目标检测与追踪

每帧经过 YOLO 检测后，接入 **ByteTrack** 追踪算法，为画面中每个物体分配持久化 ID。

输出格式：`[Object_ID, 类别, 置信度, x, y, w, h]`

例如：`ID_1_Person`、`ID_2_Cat`、`ID_3_Chair`

---

### 2. 对象状态管理

后端内存中维护以 `Object_ID` 为键的字典，记录每个物体的实时状态：

```python
active_objects = {
    "ID_1_Person": {
        "history_boxes": [...],   # 最近若干帧的坐标轨迹
        "frame_buffer": [...],    # 最近若干帧的裁剪画面
        "status": "IDLE" | "MOVING",
        "last_vlm_time": 1713000000
    }
}
```

---

### 3. 动作判定

对每个 ID，基于过去 0.5 秒（约 5 帧）的历史坐标做物理计算：

- **中心点位移（Velocity）**：中心点 X、Y 移动超过阈值，判定为位移运动
- **边界框形变率（Deformation）**：宽高比例变化超过阈值，判定为姿态变化

任意一项超过阈值，将该 ID 状态标记为 `MOVING`，进入下一步。

---

### 4. 动态裁剪（Dynamic RoI Crop）

状态为 `MOVING` 的物体，提取其过去若干帧的最大外接矩形框，向外扩展 20%–30% 作为上下文 Padding，在原始帧上用 OpenCV 裁剪出该区域，作为送审画面。

---

### 5. 送审 VLM

将裁剪后的连续帧片段连同结构化提示一起发给 VLM：

**系统提示示例：**

> 家庭智能环境监控系统报告：环境追踪引擎检测到【物体 ID_2 (类别: 猫)】在过去的 1 秒内发生了显著动作变化。
>
> 输入为包含该物体动作过程的连续 5 帧局部裁剪画面。
>
> 任务：请描述该物体刚刚做了什么动作？它和预定义的哪个事件场景最匹配？

---

## VLM 推理加速：VisPCO

动态裁剪已经减少了送入 VLM 的视觉 token 数量，但 VLM 推理延迟仍是系统吞吐的瓶颈。可引入 **VisPCO**（视觉令牌剪枝配置优化）进一步降低每次推理的计算成本。

**原理**：VisPCO 训练一个Ratio Predictor，在给定计算预算 $B$ 的条件下，自动预测 VLM 每个 Transformer 层应保留多少比例的视觉 token，使推理 FLOPs 降低的同时性能损失最小。VLM 本身权重不做任何修改。

---

## 乱序处理

每个 Object_ID 在状态字典中维护一个 `vlm_busy` 标志。当某个 ID 的 VLM 调用尚未返回时，若该 ID 再次触发动作判定，直接跳过，不发起新的调用。VLM 返回后清除 `vlm_busy`，恢复接收新片段。各 Object_ID 独立管理自身标志，互不影响。
